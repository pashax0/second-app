## Назначение

Одно рабочее место владельца магазина. Ежедневный цикл:
принять товар → сфоткать → завести в каталог → собрать дроп → опубликовать →
обработать заказы → повторить.

## Роли

Одна роль — **owner**. Multi-admin, RBAC, модерация — не в scope.
Админка не деплоится на хостинг: запускается локально с prod env.

## Функциональные области

### Товары (каталог)

- **Цель:** хранить фонд того, что может появиться на витрине.
- **Операции:** создать (фото + бренд + размер + замеры + цена), удалить черновик, редактировать существующий.
- **Границы:** после попадания в активный дроп базовые поля товара фиксируются. Цена может быть переопределена на уровне `drop_items` при повторном добавлении в следующий дроп.

### Дропы

- **Цель:** собрать текущую витрину из 15–20 товаров и опубликовать.
- **Операции:** собрать черновик, опубликовать, архивировать, смотреть аналитику (продажи, выручка, статусы товаров).
- **Границы:** одновременно активен ровно один дроп. Публикация — атомарная операция (архивировать старый + перевести товары `available`).

### Заказы

- **Цель:** обработать COD-заказы от создания до доставки.
- **Операции:** список (new / confirmed / shipped / delivered / cancelled), детали, смена статуса, контакт покупателя.
- **Границы:** платёжной интеграции нет — только COD. CRM / трекинг-номера / SMS-оповещения — не в scope MVP.

### Резервации

- **Цель:** видеть активные корзины (что занято и когда освободится).
- **Операции:** список активных резерваций, принудительное истечение.

### Пользователи

- **Цель:** просмотр зарегистрированных, индивидуальный таймер корзины (доверенному — дольше, новому — меньше; правило из business.md).
- **Операции:** список, per-user override таймера.

### Настройки

- **Цель:** глобальный таймер корзины по умолчанию.

## Сценарии

**Ежедневный дроп:**
1. Принять новую партию
2. Для каждой вещи: фото, замеры → создать товар в статусе `draft`
3. Собрать дроп из новых `draft`-товаров + нераспроданных из прошлого
4. Опубликовать — старый дроп архивируется, его нераспроданные товары возвращаются в `draft`
5. В течение дня: обрабатывать входящие заказы (confirm → ship)

**Ретроспектива прошлого дропа:**
- Открыть дроп → посмотреть, что продано, выручку, какие товары «зависли» (withdrawn)

## Non-goals

- Multi-tenant / SaaS
- Платёжные интеграции (MVP — COD only)
- Публичный аналитический дашборд
- Push-кампании из UI (отдельная инфраструктура)
- Публичный хостинг админки

## Состояние реализации

| Область                                        | Статус     |
|------------------------------------------------|------------|
| Товары: list / create / delete draft           | есть       |
| Товары: edit                                   | нет        |
| Дропы: list / create / publish                 | есть       |
| Дропы: publish — атомарность (Postgres RPC)    | нет        |
| Дропы: edit / re-order items                   | нет        |
| Аналитика дропа (DropDetail)                   | есть       |
| Заказы                                         | нет        |
| Резервации (отдельная страница)                | нет        |
| Пользователи                                   | нет        |
| Настройки (глобальный таймер)                  | нет        |

Привязка к коду — через `apps/admin/src/pages/*`. Здесь не дублируем имена файлов, чтобы roadmap не устаревал при переименованиях.

## Roadmap

В порядке приоритета (обсуждаемый):

1. **Orders page** — без этого магазин не операбелен за пределами Instagram-DM.
2. **Атомарная публикация дропа** — вынести multi-step UPDATE в Postgres RPC/транзакцию.
3. **Product edit** — редактирование цен и метаданных.
4. **Settings + per-user timer** — реализация правила из business.md.
5. **Users page** — список и per-user overrides.
